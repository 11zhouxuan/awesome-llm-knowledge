# BookRAG: 层次结构感知的索引式 RAG

> 论文: [BookRAG: A Hierarchical Structure-aware Index-based Approach for Retrieval-Augmented Generation on Complex Documents](https://arxiv.org/pdf/2512.03413)
> 作者: Shu Wang, Yingli Zhou, Yixiang Fang
> 日期: 2025-12-04

## 1. 研究动机

当 RAG 系统处理**复杂文档**（如书籍、技术手册）时，传统方法面临两个核心挑战：

1. **文档层次结构丢失:** 扁平化的 chunk 分割忽视了章节、节、小节的层次关系
2. **跨段落上下文断裂:** 独立的文本块无法维持内容元素之间的语境关系

## 2. 核心方法

### 2.1 层次索引框架（Hierarchical Indexing Framework）

构建保留文档原生层次结构的索引：

```
Book
├── Chapter 1
│   ├── Section 1.1
│   │   ├── Subsection 1.1.1
│   │   └── Subsection 1.1.2
│   └── Section 1.2
├── Chapter 2
│   └── ...
```

**核心思想:** 索引不仅存储内容，还存储内容之间的结构关系（父子、前后、引用）。

### 2.2 基于索引的检索（Index-based Retrieval）

利用文档结构改善相关性排序：
- 不是扁平地对所有 chunks 做相似度搜索
- 而是先定位结构上最相关的区域，再在区域内精确检索
- 利用层次关系提供额外的相关性信号

### 2.3 上下文保留（Context Preservation）

在检索和生成阶段维持文档元素间的上下文关系：
- 检索到的 chunk 携带其结构位置信息
- 相邻和父级内容作为上下文一并提供
- 保持了原文的逻辑流和叙事连贯性

## 3. 技术架构

### 3.1 索引构建

| 步骤 | 操作 |
|------|------|
| 结构解析 | 识别文档的层次结构（标题级别、编号） |
| 节点创建 | 为每个结构单元创建索引节点 |
| 关系映射 | 建立节点间的层次和引用关系 |
| 内容索引 | 对每个节点的内容创建检索索引 |

### 3.2 检索流程

1. 查询分析 → 确定需要的信息类型和粒度
2. 结构导航 → 基于层次索引定位相关区域
3. 精确检索 → 在定位区域内检索具体内容
4. 上下文扩展 → 补充结构上下文

### 3.3 生成增强

- 检索结果按文档原始结构组织呈现给 LLM
- 保持章节和段落的逻辑顺序
- 提供层次位置标注帮助 LLM 理解内容在全文中的位置

## 5. 主要贡献

1. **结构感知索引:** 保留文档层次结构的索引方法
2. **高效结构化检索:** 利用文档结构提升检索精度
3. **上下文连贯性:** 生成阶段维持文档元素间的语境关系

## 6. 与其他方法的比较

| 方法 | 结构感知 | 层次索引 | 上下文保留 |
|------|----------|----------|-----------|
| 标准 RAG | ✗ | ✗ | ✗ |
| RAPTOR | 部分（聚类） | ✗ | 部分 |
| PageIndex | ✓ | ✓ | ✓ |
| BookRAG | ✓ | ✓ | ✓ |

BookRAG 与 PageIndex 的思路相近（都强调文档结构），但 BookRAG 更聚焦于**索引构建**层面的结构保留，而 PageIndex 聚焦于**推理时导航**。

## 8. 核心洞察

BookRAG 体现了一个重要的趋势——**结构是被低估的检索信号**：

1. **文档不是平坦的文本序列** ——书籍、手册、法规都有精心设计的层次结构，这些结构本身包含了作者对信息组织的意图。忽视结构等于丢弃了"免费的"元数据。

2. **检索粒度应该是动态的** ——有些问题需要一个段落，有些需要整章的概览。层次索引允许在不同粒度间灵活切换。

3. **上下文不仅是"相邻文本"** ——在层次文档中，上下文包括父级标题、同级章节、交叉引用等结构关系。

**局限性:**
- 强依赖文档具有明确的层次结构
- 对扁平化文本（如聊天记录、社交媒体）不适用
- 结构解析本身可能引入错误
- 详细的实验数据有限（PDF 内容提取受限）

**与同方向工作的互补:**
- 与 PageIndex 的推理式导航可以结合——BookRAG 构建结构化索引，PageIndex 的推理机制在索引上导航
- 与 GlobalRAG 互补——BookRAG 解决单文档深度理解，GlobalRAG 解决跨文档聚合
