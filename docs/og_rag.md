# OG-RAG: Ontology-Grounded Retrieval-Augmented Generation

> 论文: [OG-RAG: Ontology-Grounded Retrieval-Augmented Generation For Large Language Models](https://arxiv.org/html/2412.15235v1/)
> 作者: Kartik Sharma, Peeyush Kumar, Yunqing Li
> 机构: Microsoft Research, Seattle
> 日期: 2024-12

## 1. 研究动机与问题

传统 RAG 系统依赖基于语义相似度的向量检索，存在以下问题：
- 检索到的上下文缺乏领域结构约束，容易引入噪声
- 无法保证生成内容与领域本体的一致性
- 事实归因困难，难以追溯回答来源

OG-RAG 提出用领域本体（Ontology）锚定检索过程，通过超图（Hypergraph）表示和优化算法，生成精确的、概念上有根基的上下文。

## 2. 核心方法

### 2.1 超图构建（Hypergraph Construction）

将文档映射为超图结构：
- 每个超边（Hyperedge）包含基于本体约束的事实知识簇
- 将嵌套的事实块展平为超边
- 创建领域知识的结构化表示

与传统图谱（每条边只连接两个节点）不同，超图的每条超边可连接多个节点，能建模多维关系。

### 2.2 基于超图的检索（Hypergraph-Based Retrieval）

1. 基于语义相似度（text-embedding-3-small）识别与查询相关的超节点
2. 使用贪心优化算法选择覆盖相关节点的最小超边集合
3. 在 matroid 约束下贪心算法可提供最优解

### 2.3 上下文生成

- 将检索到的超边格式化为结构化事实
- 以领域特定词汇供给 LLM
- 查询时不需要额外的 LLM 调用

### 2.4 本体形式化

本体定义为三元组，关联实体与属性关系。系统通过本体约束确保知识表示的语义一致性。

## 3. 实验评估

### 3.1 数据集

| 数据集 | 描述 |
|--------|------|
| Agriculture | 85 篇大豆/小麦种植专有文档 |
| News | 149 篇长篇新闻文章（Multi-hop RAG） |

### 3.2 对比基线

- RAG（基于向量相似度检索）
- RAPTOR（层次聚类+摘要）
- GraphRAG（LLM 提取知识图谱）

### 3.3 核心结果

**上下文召回（Context Recall）:**

| 数据集 | OG-RAG | RAPTOR | RAG |
|--------|--------|--------|-----|
| Soybean | **0.84** | 0.54 | 0.22 |
| Wheat | **0.95** | 0.85 | - |
| News | **0.82** | 0.82 | - |

**回答正确性（Answer Correctness）:**

| 数据集 | OG-RAG | RAPTOR |
|--------|--------|--------|
| Soybean | **0.48** | 0.34 |
| Wheat | **0.62** | 0.59 |

### 3.4 效率指标

| 指标 | OG-RAG | RAPTOR |
|------|--------|--------|
| 预处理时间 | 29.61-655.15s | 61.56-1513.57s |
| 查询时间 | 3.75-4.12s | - |
| 上下文大小 | 10-655 tokens | - |

### 3.5 归因研究（16名参与者用户研究）

| 指标 | OG-RAG | RAG |
|------|--------|-----|
| 事实验证时间 | 43.5±18.08s | 61.15±28.48s |
| 支持度评分 | 3.46±0.19 | 2.67±0.30 |

## 4. 关键性能数据

- 事实召回准确率提升 **55%**
- 回答正确性提升 **40%**
- 响应归因速度加快 **30%**
- 事实推理准确率提升 **27%**

## 5. 复杂度分析

**时间复杂度:**
- 最小嵌套: O(|Ontology| × |Factual blocks|)
- 最大嵌套: O(|Factual blocks| × |Ontology| × log|Ontology|)

**空间复杂度:** 与超边数量成正比

**LLM 调用:** 预处理阶段 (|Documents| + |Ontology|) / Context_Size，查询时无需 LLM 调用

## 6. 主要贡献

1. **基于事实的上下文检索:** 通过超图形式化和优化检索将领域本体集成到检索流程
2. **事实推演:** 支持领域适配的事实推理
3. **上下文归因:** 提供精确的事实级别响应溯源

## 7. 局限性与未来方向

- 需要预定义领域本体（核心局限）
- 本体学习方法作为未来工作
- 部分数据集上回答相关性略低（可能因更宽泛的上下文）
- 建议进一步做领域特定优化

## 8. 适用场景

- 医疗工作流
- 法律分析
- 农业决策
- 新闻调查
- 需要强事实约束的领域

## 9. 核心洞察

OG-RAG 的关键创新在于用本体约束将"语义相似"提升为"语义正确"——不是找到最相似的内容，而是找到在领域知识框架内最正确的事实。超图结构天然适合表示多维关系，而查询时零 LLM 调用的设计使其在生产环境中具有成本优势。代价是前期本体构建的投入，适合本体相对稳定的规范化领域。
